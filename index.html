Why teams pick RuneFlow

Anchor-based reliability

Each click captures a small screenshot “anchor”. On replay we match via OpenCV (color, grayscale, edges + scale sweep) for robust targeting.

Adaptable by design

Re-anchor at the cursor, tweak confidence/timeout, set uniqueness margins, and constrain searches with lock-to-window for extra speed.

Editor that respects time

Reorder, duplicate, delete, insert waits, edit per-step delays, and export a portable .py runner. No baggage, no bloat.

Optional VLM assist

When the UI really moves, a configurable Vision-LLM can suggest a click, then mini-verify before acting. You stay in control with a switch.

Fast & local

OpenCV matching runs locally. Runner JSON is normalized with relative assets so playback is portable across machines.

Pragmatic controls

Jitter, slow/fast delay scaling, stubborn search caps, recovery JSON, ESC to stop, and keyboard hold-to-confirm double-clicks for flaky UIs.

How it works

  1. Record: Click/keys are captured with tiny anchor images. F8 stops recording.
  2. Refine: Reorder, insert waits, re-anchor, tweak confidence & timeouts.
  3. Run: Replay with jitter + delay scaling. Optional recovery flows.
  4. Export: One click to a standalone Python runner you can schedule anywhere.
